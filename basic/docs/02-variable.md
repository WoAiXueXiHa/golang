# Go 变量

## 这一章要记住什么

这一章主要讲四个点：

- 变量只声明不初始化，会使用对应类型的零值。
- Go 有多种变量声明方式。
- `:=` 只能在函数内部使用。
- 多变量赋值可以很方便地交换两个值。

---

## 1. 零值

代码里：

```go
var a int
var b bool
var c float64
var d string
```

这些变量只声明，没有手动赋值。

Go 会自动给它们对应类型的零值：

```text
int     -> 0
bool    -> false
float64 -> 0
string  -> ""
```

```text
声明变量
   |
   v
有没有手动赋值？
   |
   +-- 有：使用给定值
   |
   +-- 没有：使用类型零值
```

## 总结一下

Go 里的变量不会处在“未初始化的随机状态”。

只要声明了变量，它一定有一个确定的零值。

---

## 2. 变量声明方式

代码里展示了三种常见写法：

```go
var num int = 20
str := "golang"
var val = 2.10
```

区别是：

- `var num int = 20`：类型和值都写清楚。
- `str := "golang"`：短变量声明，类型由右边推导。
- `var val = 2.10`：用 `var` 声明，但类型也由右边推导。

```text
var num int = 20
      |
      v
明确告诉编译器：num 是 int

str := "golang"
      |
      v
编译器根据右边推导：str 是 string
```

## 总结一下

平时函数内部写临时变量，`:=` 很常用。

需要强调变量类型、声明包级变量，或者先声明后赋值时，用 `var` 更合适。

---

## 3. 多变量声明

代码里：

```go
var v1, v2 = 100, "ll"
n1, n2 := 1.0, 200
```

Go 支持一次声明多个变量，而且类型可以不一样。

```text
v1 -> 100  -> int
v2 -> "ll" -> string

n1 -> 1.0 -> float64
n2 -> 200 -> int
```

## 总结一下

Go 的多变量声明很适合处理多个返回值，也适合写简单的交换逻辑。

---

## 4. 交换两个变量

代码里：

```go
val1, val2 = val2, val1
```

这行可以直接交换两个变量，不需要额外的临时变量。

```text
交换前：
val1 = 100
val2 = 0

执行：
val1, val2 = val2, val1

交换后：
val1 = 0
val2 = 100
```

## 总结一下

Go 支持多变量同时赋值，右边会先计算，再统一赋给左边。

所以交换变量时不会出现覆盖问题。

---

## 易错点

1. `:=` 只能在函数内部使用，不能用于包级变量声明。
2. 变量声明后必须使用，否则编译报错。
3. `var d string` 的零值是空字符串，不是 `nil`。
4. 多变量赋值是同时完成的，适合交换变量。

---

## 快问快答

### Q1：Go 变量不初始化会怎么样？

答：

会使用类型零值。比如 `int` 是 `0`，`bool` 是 `false`，`string` 是空字符串。

### Q2：`var` 和 `:=` 有什么区别？

答：

`var` 可以用在函数内和函数外，`:=` 只能在函数内部。`:=` 更适合声明临时变量。

### Q3：Go 怎么交换两个变量？

答：

可以直接写 `a, b = b, a`。

---

## 一句话总结

Go 变量声明很直接：有零值、有类型推导、有短声明，也支持多变量同时赋值。
