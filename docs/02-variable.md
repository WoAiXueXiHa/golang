# 02-variable：变量不是语法题，是 Go 的默认值和作用域习惯

这个目录最重要的是三件事：**零值、短变量声明、多重赋值**。代码不复杂，但它们会贯穿所有 Go 后端代码。你以后写 `user, err := repo.Find(...)`、`var cfg Config`、`a, b = b, a`，都来自这里。

## 关键代码怎么看

`testZero()` 演示零值：

```go
var a int
var b bool
var c float64
var d string
fmt.Println(a, b, c, "-", d, "-")
```

Go 不会让局部变量处在“随机垃圾值”状态。只声明不赋值，也会得到明确零值：数字是 `0`，bool 是 `false`，string 是 `""`。这是 Go 比 C/C++ 更安全的地方。

但要小心：零值安全不等于所有零值都能直接写入。比如 `var m map[string]int` 是 nil map，读可以，写会 panic；`var s []int` 是 nil slice，`append` 可以，但 `s[0]` 不行。

`testVar()` 里有几种声明方式：

```go
var num int = 20
str := "golang"
var val = 2.10
var v1, v2 = 100, "ll"
n1, n2 := 1.0, 200
```

复习时不用背“几种写法”，直接记使用场景：

- 函数内部有初始值，优先用 `:=`。
- 包级变量、需要显式类型、先声明后赋值，用 `var`。
- 多返回值接收，几乎天天用 `:=`，比如 `val, ok := m[key]`、`data, err := os.ReadFile(...)`。

`exangeVal()` 演示多重赋值：

```go
val1, val2 = val2, val1
```

Go 会先计算右边，再统一赋给左边，所以不需要临时变量。注意这个函数名拼错了，应该是 `exchangeVal`。更重要的是，它只交换了函数参数副本，调用方的值不会变。

## 必须掌握的点

**1. 零值是 Go 的基础设计。**  
结构体初始化时经常只填关键字段，其余依赖零值。比如 `User{Name: "tom"}` 中 `ID` 默认是 `0`。

**2. `:=` 只能在函数内部使用。**  
包级别不能写 `name := "go"`，必须写 `var name = "go"`。

**3. `:=` 左边至少要有一个新变量。**  
这就是为什么下面这种写法成立：

```go
file, err := os.Open(path)
n, err := file.Read(buf)
```

第二行里 `n` 是新变量，`err` 是重新赋值。

**4. Go 只有值传递。**  
`exangeVal(100, 0)` 交换的是参数副本。以后你想让函数修改外部变量，要传指针，或者返回新值。

## 用一个形象例子理解

零值像酒店房间的默认状态：没人入住时，灯关着、空调关着、床是空的。你不用担心房间里随机出现陌生人。

但 nil map 像“还没开张的前台登记本”：你可以问“有没有 101 房客”，答案是没有；但你不能直接往不存在的登记本上写名字，得先 `make` 一本登记本。

## 和 Go 后端开发的关系

变量声明习惯直接影响代码味道：

- `err` 通常就近声明，就近处理，不要飘到很远的作用域。
- 配置结构体常依赖零值提供默认行为。
- map 读取要写 `v, ok := m[key]`，不能只看零值。
- 函数如果要修改调用方数据，优先明确传指针或返回结果，不要让副作用藏起来。

## 更像工程代码的写法

- `exangeVal` 改成 `exchangeVal`。
- `fmt.Printf` 后最好补换行，否则输出容易挤在一起。
- 打印空字符串时用 `%q` 更清楚：`fmt.Printf("d=%q\n", d)`。
- 如果只是演示变量类型，输出可以更有标签，避免复习时看不出每个值对应谁。

## 复习时问自己

1. **`var s string` 的零值是什么？**  
   空字符串 `""`，不是 nil。

2. **`var m map[string]int` 能不能直接 `m["a"] = 1`？**  
   不能，会 panic。需要 `make(map[string]int)` 或 map 字面量。

3. **`:=` 为什么不能放在函数外？**  
   因为它是短变量声明，只用于函数内部语句块。

4. **`exangeVal` 为什么不能改变调用方的变量？**  
   因为参数是值传递，函数里拿到的是副本。
